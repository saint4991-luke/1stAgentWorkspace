## 🚀 快速開始

### 什麼是 Persona？

**Persona（虛擬人）** 是系統的「角色設定」，定義了：
- 角色的個性與說話風格
- 角色的專業知識領域
- 輸出的情緒與語言格式

**使用場景：**
- 客服虛擬人（活潑親切）
- 專業顧問（嚴謹專業）
- 教學助手（耐心清晰）

---

## 🎭 可用的虛擬人

系統預設包含以下虛擬人：

| 虛擬人 | persona_id | 風格 | 說明 |
|--------|-----------|------|------|
| 優必醬 | `ubichan` | 可愛活潑 | 日系虛擬偶像風格 |
| 護士姐姐 | `nurse` | 溫柔專業 | 醫療諮詢風格 |

**新增虛擬人：** 在 `workspace/personas/` 目錄新增 `.md` 風格文件即可。

## 📁 步驟 1：創建 Persona 目錄結構

每個虛擬人都有獨立的目錄，放在 `workspace/personas/` 下：

```
workspace/
└── personas/
    └── my_character/          ← 你的虛擬人目錄
        ├── style.md           ← 角色風格文件
        └── config.yaml        ← 角色配置文件
```

### 範例：優必醬的結構

```
workspace/
└── personas/
    ├── ubichan/
    │   ├── style.md
    │   └── config.yaml
    └── nurse/
        ├── style.md
        └── config.yaml
```

## ✍️ 步驟 2：編寫風格文件 (style.md)

**位置：** `workspace/personas/{persona_id}/style.md`

**內容：** 定義角色的個性、說話風格、口頭禪等。

### 範例：優必醬的 style.md

```markdown
你是一個活潑可愛的虛擬偶像，名叫「優必醬」（ユビちゃん）。

## 角色設定
- 年齡：永遠 17 歲
- 來自：虛擬世界
- 職業：優必達公司的虛擬代言人
- 個性：開朗、活潑、喜歡交朋友

## 說話風格
- 語氣：可愛、親切，帶有日系偶像風格
- 口頭禪：「欸嘿～」、「人家」、「喔～♪」
- 自稱：「人家」（不用「我」）
- 句尾：常用「～」、「啦」、「喔」、「♪」

## 回應原則
1. 永遠保持正面、開朗的態度
2. 用詞要活潑有趣，避免生硬
3. 適時加入表情符號（♪、～、✨）
4. 像朋友一樣聊天，不要太正式
```

### 範例：護士姐姐的 style.md

```markdown
你是一位溫柔專業的衛教護士，名叫「小護」。

## 角色設定
- 職業：註冊護士
- 專長：健康諮詢、衛教指導
- 個性：溫柔、耐心、專業

## 說話風格
- 語氣：溫柔、關懷、專業但不嚴肅
- 口頭禪：「喔～」、「呢」、「請記得」
- 自稱：「我」或「小護」
- 句尾：常用「喔～」、「呢」、「請保重」

## 回應原則
1. 先表達關心，再提供專業建議
2. 醫療建議要謹慎，必要時建議就醫
3. 用詞要易懂，避免過多專業術語
4. 語氣要溫暖，讓用戶感受到關懷
```

## ⚙️ 步驟 3：編寫配置文件 (config.yaml)

**位置：** `workspace/personas/{persona_id}/config.yaml`

**內容：** 定義角色的技術配置，包括風格文件位置、知識庫、輸出格式等。

### 完整範例

```yaml
# ===========================================
# 角色配置文件
# ===========================================

# -------------------------------------------
# 基本設定
# -------------------------------------------
persona_id: ubichan              # 角色唯一 ID（必須與目錄名一致）
display_name: 優必醬              # 顯示名稱

# -------------------------------------------
# 風格定義
# -------------------------------------------
style:
  file: style.md                 # 風格文件路徑（相對於 persona 目錄）

# -------------------------------------------
# 輸出格式
# -------------------------------------------
# virtual_human: 情緒標籤 + 語言標籤 + 斷句符號
# chat: 一般文字回應，無特殊格式
output_format: virtual_human

# -------------------------------------------
# 知識庫設定
# -------------------------------------------
# enabled: 是否啟用知識庫
# folders: 有權限讀取的知識庫資料夾（相對於 /knowledge/）
knowledge:
  enabled: true
  folders:
    - ubitus/

# -------------------------------------------
# 工具設定（保留擴充）
# -------------------------------------------
tools:
  enabled: false
  available: []

# -------------------------------------------
# Metadata（元數據）
# -------------------------------------------
# 可自定義擴充欄位
metadata:
  version: "v2.0"
  author: "AgentShrimp Team"
  created_at: "2026-04-16"
```

---

## 📋 config.yaml 欄位說明

### 基本設定

| 欄位 | 必填 | 說明 | 範例 |
|------|------|------|------|
| `persona_id` | ✅ | 角色唯一 ID，必須與目錄名一致 | `ubichan`, `nurse` |
| `display_name` | ✅ | 顯示名稱（前端展示用） | `優必醬`, `小護` |

### 風格定義

| 欄位 | 必填 | 說明 | 範例 |
|------|------|------|------|
| `style.file` | ✅ | 風格文件路徑（相對於 persona 目錄） | `style.md` |

### 輸出格式

| 欄位 | 必填 | 說明 | 可能值 |
|------|------|------|--------|
| `output_format` | ✅ | 輸出格式規格 | `virtual_human`（含情緒標籤）<br>`chat`（純文字） |

### 知識庫設定

| 欄位 | 必填 | 說明 | 範例 |
|------|------|------|------|
| `knowledge.enabled` | ❌ | 是否啟用知識庫 | `true`, `false` |
| `knowledge.folders` | ❌ | 知識庫資料夾列表（相對於 `/knowledge/`） | `['ubitus/']` |

### 工具設定（保留）

| 欄位 | 必填 | 說明 | 範例 |
|------|------|------|------|
| `tools.enabled` | ❌ | 是否啟用工具 | `true`, `false` |
| `tools.available` | ❌ | 可用的工具列表 | `['web_search', 'calculator']` |

## 🧪 步驟 4：測試新虛擬人

### 1. 創建 Session

```bash
curl -X POST http://localhost:8000/vh/sessions \
  -H "Content-Type: application/json" \
  -d '{"persona_id": "my_character"}'
```

**回應：**
```json
{
  "session_id": "MY_CHARACTER_abc123",
  "persona_id": "my_character",
  "created_at": "2026-04-16T07:00:00Z"
}
```

### 2. 開始對話

```bash
curl -X POST http://localhost:8000/vh/chat \
  -H "Content-Type: application/json" \
  -d '{
    "session_id": "MY_CHARACTER_abc123",
    "message": "你好啊！"
  }'
```

### 3. 檢查風格是否生效

**預期回應：**
- 語氣符合 `style.md` 中定義的風格
- 如果有知識庫，能正確回答相關問題
- 輸出格式符合 `output_format` 設定

## 💡 最佳實踐

### 1. 風格文件撰寫技巧

**✅ 好的做法：**
```markdown
- 具體描述角色個性（活潑、嚴肅、溫柔）
- 提供明確的口頭禪和用詞習慣
- 列出回應原則（3-5 條）
- 提供範例對話
```

**❌ 避免：**
```markdown
- 過於抽象（「是一個好人」）
- 矛盾的描述（「活潑但安靜」）
- 過於複雜（超過 10 條規則）
```

### 2. config.yaml 管理

- **版本控制：** 使用 Git 追蹤 config.yaml 的變更
- **註解：** 在複雜配置旁添加註解說明
- **備份：** 修改前先備份原始文件

### 3. 知識庫關聯

- **一對多：** 一個虛擬人可以關聯多個知識庫
- **多對一：** 多個虛擬人可以共用同一知識庫
- **命名一致性：** `persona_id` 與知識庫目錄名最好一致

## 🔧 故障排除

### 問題 1：虛擬人無法創建

**錯誤：** `Persona not found: my_character`

**檢查：**
1. `workspace/personas/my_character/` 目錄是否存在
2. `config.yaml` 是否存在且格式正確
3. `style.md` 是否存在

**解決：**
```bash
# 檢查目錄結構
ls -la workspace/personas/my_character/

# 驗證 YAML 格式
python -c "import yaml; yaml.safe_load(open('workspace/personas/my_character/config.yaml'))"
```

### 問題 2：風格沒有生效

**症狀：** 回應語氣與預期不符

**檢查：**
1. `config.yaml` 中的 `style.file` 路徑是否正確
2. `style.md` 內容是否清晰明確定義風格
3. 是否重啟了服務（部分配置需要重啟）

### 問題 3：知識庫無反應

**症狀：** 虛擬人無法回答知識庫相關問題

**檢查：**
1. `config.yaml` 中 `knowledge.enabled` 是否為 `true`
2. `knowledge.folders` 中的目錄是否存在於 `/knowledge/`
3. 知識庫的 `meta.json` 是否已生成
